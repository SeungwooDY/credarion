## ERP Record Deduplication

**Problem:** Uploading the same GRN file twice inserted duplicate `ERPRecord` rows. Suppliers were safe (upsert logic), but data rows had no guard.

**Solution:** On ingestion, fetch all existing `(supplier_id, po_number, material_number, grn_number)` tuples for the org in one query, build an in-memory set, and skip rows whose key already exists.

- Single-query approach: ~millisecond overhead regardless of table size
- Also prevents duplicates within the same file (same key appears twice)
- `GRNIngestionResult` and API response now include `rows_duplicate` count
- UI shows "Duplicates skipped: N" when > 0

**Files changed:**
- `backend/app/ingestion/grn_ingestor.py` — added dedup set, `rows_duplicate` field
- `backend/app/routers/erp.py` — `GRNIngestionResponse` includes `rows_duplicate`
- `backend/app/static/upload.html` — displays duplicate count in result

## SSE Progress Streaming for GRN Upload

**Problem:** GRN upload takes ~60s for 6,648 rows with no feedback — users don't know if it's working.

**Solution:** New `POST /api/v1/erp/upload-stream` endpoint using `StreamingResponse` with `text/event-stream`:
- Ingestion runs in a background thread
- Progress callback fires ~20 times during processing
- Phases: `reading` → `suppliers` → `dedup` → `ingesting` (0-100%) → `saving` → `done`
- UI shows a progress bar with phase labels
- Original `POST /upload` endpoint preserved for programmatic use

**Files changed:**
- `backend/app/ingestion/grn_ingestor.py` — added `on_progress` callback parameter
- `backend/app/routers/erp.py` — added `upload-stream` endpoint
- `backend/app/static/upload.html` — progress bar, `uploadGRN()` uses streaming endpoint

## Supplier Auto-Load on Org Select

**Problem:** Supplier dropdown in Step 3 only populated after GRN upload. Refreshing the page or returning later showed "Upload GRN first" even though suppliers existed in the DB.

**Solution:** Call `loadSuppliers()` on page load (when an org is auto-selected) and on org dropdown change. Shows "No suppliers yet — upload a GRN file first" only when the DB query returns empty.
