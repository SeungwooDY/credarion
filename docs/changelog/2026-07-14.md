# Changelog — 2026-07-14

## Security fix: dashboard cross-tenant supplier leak

### Bug
`GET /api/v1/reconciliation/dashboard` (`backend/app/routers/reconciliation.py`)
auto-selected an organization with `db.query(Organization).first()` when no
`org_id` was supplied — returning the *first org in the entire database*
regardless of the caller's account. The router-level `enforce_org_scope`
guard is a no-op when `org_id` is absent, so nothing caught it.

Effect: an account with no organization of its own (e.g. the `noorg` test
account) saw another tenant's suppliers — surfaced as a "0/209 suppliers
reconciled" tile where it should have read 0/0. Any authenticated customer
without an explicit `org_id` could read another tenant's dashboard totals.

### Fix
`dashboard_overview` now takes `user: User = Depends(get_current_user)` and
scopes the auto-select through `accessible_org_ids(db, user)` — the same
account-scoping helper already used by `/summary`, `/runs`, `/results`, and
`/export`. Accounts with no org now correctly get an empty dashboard (0/0).
Superusers remain unrestricted. Explicitly-passed `org_id` values are still
validated by `enforce_org_scope` as before.

### Tests (`backend/tests/test_reconciliation_tenant_isolation.py`)
- `test_dashboard_autoselect_scoped_to_account` — Account A's dashboard never
  auto-selects Account B's supplier.
- `test_dashboard_empty_for_account_without_org` — a no-org account gets `[]`.

Full backend suite: 281 passed.
