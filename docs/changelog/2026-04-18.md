Two parallel workstreams merged into main today. Combined, they complete the data pipeline end-to-end: file upload → ingestion → reconciliation → results API.

### New Files
| File | Purpose |
|------|---------|
| `backend/app/ingestion/grn_ingestor.py` | SGWERP GRN CSV ingestion pipeline. Resolves Chinese/English column headers via static alias map, normalizes POs/quantities/prices/currency/VAT, upserts suppliers, bulk inserts `ERPRecord` rows. |
| `backend/app/routers/erp.py` | `POST /api/v1/erp/upload` — multipart GRN file upload. `POST /api/v1/erp/upload-stream` — same but with SSE progress events. |

| `backend/tests/test_grn_ingestion.py` | 28 tests covering column resolution, VAT/currency parsing, end-to-end ingestion with in-memory SQLite. |

### Key Design Decisions
- **Static alias map for GRN columns** — unlike supplier statements (which need header detection + AI mapping), the GRN export has consistent column names. A simple dict lookup suffices.
- **Supplier auto-creation** — uploading a GRN file upserts all unique vendors as `Supplier` records, so they're immediately available for statement uploads.
- **SSE progress streaming** — `upload-stream` endpoint runs ingestion in a background thread, pushes progress events (~20 updates per file) via `text/event-stream`. The UI shows a progress bar with phase labels (reading → suppliers → ingesting → saving → done).

### Column Alias Fixes (post-merge)
The real GRN CSV uses short English headers (`po`, `pn`, `grn_accept`, `AMOUNT`, `grn_no`, `do_number`, `dn_no`, `po_cur`, `po_vat`). These were added to `GRN_COLUMN_ALIASES` after the initial merge since Agent A's alias map only had Chinese headers + long English names.

## Agent B: Reconciliation Engine (Phase 2)

### Migration
- **`0003_reconciliation_engine.py`** — Creates `reconciliation_runs` (execution tracking) and `reconciliation_config` (per-org tolerance settings). Extends `reconciliation_results` with `run_id`, `amount_delta`, `discrepancy_type`, `confidence`, `resolved_by`, `resolved_at`, `match_details`.

### New Models (in `models.py`)
- **`ReconciliationRun`** — Tracks each execution: supplier, period, status, match counts, auto_match_rate, config snapshot (JSONB).
- **`ReconciliationConfig`** — Per-org settings: qty/price tolerance %, AI toggle, token budget.
- **`ReconciliationResult`** — Extended with discrepancy classification, confidence scores, resolution workflow fields.

### Reconciliation Engine (`backend/app/reconciliation/`)
| File | Purpose |
|------|---------|
| `normalization.py` | `normalize_po_for_matching()`, `normalize_material_for_matching()` — strip leading zeros, dashes, spaces, uppercase. |
| `exact_match.py` | **Layer 1:** Dict lookup on `(po_number, material_number)`. Tolerance check on qty/price. Duplicate PO+PN tiebreaker by `grn_date` proximity or `delivery_note` match. |
| `fuzzy_match.py` | **Layer 2:** Normalized PO matching. Catches leading zeros, dashes, spaces, case differences. |
| `multi_po_dn.py` | **Layer 3:** Groups by delivery note, compares aggregated qty/amounts. Handles multi-PO delivery notes without double-counting. |
| `ai_match.py` | **Layer 4:** Batches unmatched items (15/call) to Claude Haiku. Confidence >= 0.7 threshold. Graceful fallback if API unavailable. |
| `orchestrator.py` | Waterfall executor: loads ERP + statement data → runs layers 1-4 → classifies discrepancies → bulk inserts results → updates run stats. |
| `schemas.py` | Pydantic request/response models for all reconciliation endpoints. |

## Current Test Flow (validated)
1. Open `http://localhost:8000` → create org
2. Upload `data/samples/erp/March Goods Receipt Detail.xlsx - GRN.csv` → 6,648 rows ingested, ~214 suppliers created
3. Select supplier (e.g. Aoxiong) → upload `data/samples/supplier_statements/Guowei-Aoxiong March 2026 Reconciliation.xlsx - Sheet1.csv` with period `2026-03`
4. `POST /api/v1/reconciliation/run` with supplier_id + period `2026-03` → runs 4-layer matching
5. `GET /api/v1/reconciliation/results?status=discrepancy` → view flagged items
6. `PUT /api/v1/reconciliation/results/{id}/resolve` → resolve with note
7. `GET /api/v1/reconciliation/export?format=xlsx` → download report
